# Xaero 多域名填充剔除修复设计

## 目标

修复 Xaero's Minimap 与 Xaero's World Map 中只有部分顶点绕序的域名能够显示填充的问题，同时保留当前域名挖空、祖先差集填充、颜色、边界及交互的现有行为。

## 根因

`PolygonGeometry.triangulate` 保留域名 JSON 顶点的顺时针或逆时针绕序，`AreaOverlayFillResolver` 生成的普通网格和差集网格也没有强制统一绕序。Xaero 在世界地图元素渲染前开启背面剔除，小地图覆盖层也不能保证剔除处于关闭状态，而 `OverlayRenderHelper` 只管理混合、深度和着色器。因此，只有最终朝向当前屏幕正面的三角形会被提交到画面，边界线和颜色解析则不受影响。

## 方案

在两张地图共用的 `OverlayRenderHelper` 中把剔除纳入覆盖层渲染状态：

1. `beginOverlay` 捕获进入覆盖层前的 `GL_CULL_FACE` 状态。
2. 每次准备几何状态时关闭背面剔除，使域名面不依赖 JSON 顶点绕序。
3. `endOverlay` 按捕获值恢复剔除状态，避免影响 Xaero 后续图标、文字和地图绘制。
4. 保留现有线程局部状态栈，使嵌套覆盖层仍能逐层恢复各自的原始状态。

## 不采用的方案

- 强制重排三角形绕序：世界地图与小地图使用不同矩阵，差集算法还会产生新多边形，固定世界坐标绕序不能可靠代表最终屏幕朝向。
- 同时提交正反两套三角形：会增加顶点提交量，并可能在外部已关闭剔除时造成透明颜色重复叠加。

## 验证

遵照仓库要求不新增测试文件。通过源代码对照确认任意绕序均不再被剔除，检查差异只涉及共享渲染状态和计划文档，并运行 `\.\gradlew.bat build` 验证 Java 17、Fabric 1.20.4 及两个 Xaero 固定版本下的完整编译、测试和打包。
