# Xaero 覆盖层渲染修复设计

## 目标

修复 Areas Hint 与 Xaero's Minimap 25.2.0、Xaero's World Map 1.39.2 联动时的三个问题：世界地图域名几何随缩放消失或比例错误、域名填充不一致，以及小地图边线越出地图外框。

## 根因

1. 世界地图只在整批覆盖层开始时绑定一次纯色着色器；域名文字绘制会切换着色器，后续域名的填充与边线因继续使用文字着色器而丢失。文字恰好受缩放阈值控制，因此问题表现为缩小到一定程度后边线才恢复。
2. 世界地图元素中心由 Xaero 按 `mapScale` 变换，但 Areas Hint 的相对顶点没有应用相同缩放，导致多边形几何与地图缩放不一致。
3. 小地图现有逻辑只填充玩家所在位置最深层的域名，却为所有可见域名绘制边界，因此视觉上天然存在部分填充、部分不填充。
4. 圆形小地图已经对几何进行本地裁剪，方形小地图则主要依赖屏幕 scissor；后者无法稳定覆盖 Xaero 的全部矩阵变换，边线可能越出外框。

## 设计

- 在每次提交三角形或线段前恢复覆盖层需要的混合、深度和纯色着色器状态，避免文字绘制污染后续几何。
- 世界地图在域名元素局部矩阵中应用 `mapScale`；名称绘制继续使用逆缩放，从而保持文字尺寸不随地图缩放。
- 小地图先绘制所有当前高度可见域名的填充，再统一绘制所有边界，确保每个域名都有填充且边界保持清晰。
- 圆形小地图继续使用圆裁剪；方形小地图新增矩形多边形和线段裁剪。现有 scissor 保留为最后一道保护，但不再承担主要几何裁剪职责。
- 联合域名文字、最深层名称选择、世界地图悬停提示和右键域名管理入口全部保持现状。

## 验证

仓库明确要求不新增测试，因此不创建测试文件。实现后运行客户端 Java 编译、完整 Gradle 构建、`git diff --check` 和变更范围检查；游戏内行为由用户自行验证。
