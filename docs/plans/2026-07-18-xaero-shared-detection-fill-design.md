# Xaero 共享域名判定与动态填充设计

## 目标

修复 Xaero's Minimap 与 Xaero's World Map 中填充状态固定在某个域名、无法随玩家进入或离开而变化的问题，同时避免两张地图为确定玩家所在域名重复执行点与多边形判定。小地图只保留实际域名的边界和填充，不再显示 `name` 或 `surfacename` 文字；世界地图的名称、悬停信息和右键管理保持不变。

## 根因与约束

现有填充解析器在主功能开启时已经读取 `AreaChangeTracker`，但只从可变的 `AreaData` 中临时取得名称，两张地图又分别持有解析器与填充缓存。判定结果缺少明确的维度和状态版本，填充计划只能依赖“快照版本、维度、域名名称”推断状态是否变化；主功能关闭时，两张地图还会分别执行备用判定。该结构无法保证一次模组判定只驱动一次明确的填充状态转换，也容易让旧计划继续被渲染。

仓库明确要求不新增测试、直接在 main 工作区修改且不提交，因此验证只使用编译、完整构建和静态差异检查。

## 判定状态

`AreaChangeTracker` 在保留现有 `AreaData` 接口的同时，对外提供不可变判定快照。快照包含当前实际域名名称、当前维度标识和单调递增的状态版本。

- 初次进入域名、离开域名、从一个域名进入另一个域名、切换维度或重置状态时发布新快照。
- 没有发生有效状态变化时保留原版本，普通渲染帧不会制造新状态。
- 快照使用 `volatile` 发布，使渲染读取获得同一组名称、维度和版本，避免分别读取可变字段。
- Xaero 只消费已经完成的模组判定，不再次执行多边形命中。

## 共享填充缓存

小地图和世界地图改为共享一个 `AreaOverlayFillResolver`。解析器按地图维度保存最近的填充计划，缓存键包含域名快照修订号、判定状态版本、判定维度和当前实际域名名称。

- 玩家未进入任何域名时，每个实际域名使用完整三角网格。
- 玩家进入某个域名时，该域名使用空填充网格。
- 当前域名的每一级有效上级域名使用“上级域名减去上级域名与当前域名交集”的差集网格。
- 兄弟域名、未进入的子域名和其他域名保留完整填充。
- 边界不受进入状态影响。
- 世界地图查看其他维度时，该维度没有当前玩家域名，所有域名完整填充。

只有域名文件修订或模组判定状态变化时才重建差集。两张地图读取同一维度和同一状态时复用同一个不可变计划，不重复计算。

## 独立覆盖层回退

主功能关闭但 Xaero 覆盖层单独开启时，`AreaChangeTracker` 不会继续产生判定。此时共享解析器才调用现有 `AreaDetector.findAreaForXaeroOverlay`：

- 两张地图共享一次备用结果。
- 玩家坐标未变化且域名快照未更新时直接复用结果。
- 玩家移动时按模组配置的检测频率更新，不按地图渲染帧率重复判定。
- 重新开启主功能后立即恢复使用 `AreaChangeTracker` 的权威状态。

## 小地图显示

删除小地图的域名文字提交逻辑和仅为文字保存的当前域名字段。实际域名仍参与可见性筛选、动态填充、圆形或矩形裁剪以及边界绘制。世界地图继续使用 `displayName()`，因此其联合域名显示和管理能力不变。

## 验证

- 运行 `.\gradlew.bat compileClientJava` 验证客户端与 Xaero API 编译。
- 运行 `.\gradlew.bat build` 验证完整项目构建。
- 运行 `git diff --check` 检查空白和补丁格式。
- 检查修改范围不包含测试、生成输出或翻译文件。
